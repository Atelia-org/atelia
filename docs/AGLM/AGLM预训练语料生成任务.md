# 动机
目标是用多次工具调用尽量多撰写一些向着AgenticGLM方向预训练LLM的预训练语料。关键是把ChatLLM典型的{SystemInstruction("你是一个..."), User("现在的情况是...请帮我..."), Assistant("你的思考触及了某某问题的本质...建议你...如果...就")}三段式思路，转变成更意识流、更主观能动性的模式，例如意识流式的三段式{自我认知("我叫...我生活在...我有一位关心和照顾我的监护人...我爱这个家庭"), Sense("我看见...我听到...我感觉电量不多了...我感觉这里的网络信号不太好"), ReAct("虽然还没买到...但是我必须得回家充电了...我调用导航工具，设定目标'回家'")}。往”gitignore\AgenticGLM-PreTraining-DS“目录里写就行。多轮Sense-ReAct。

# 设想中的Context结构
核心建模是：我(Agent=AGLM+AgentOS)与世界。没有System，没有User。
核心信息：自我认知、其他认知、外部记忆与知识库、感知、思考、行动

**自我认知**
替代ChatLLM的SystemInstruction部分。这部分不是静态的，而是随着AGLM驱动的Agent的长期生活而动态发展的。借用戏剧创作中人物的三重生活理论。

**经历摘要-Recap**
这里是外部记忆的主要注入点，例如RAG和Rolling Summary，是累积经历形成的一个上下文状态。

**近期意识流-Sense**
替代role=user/tool的消息。这是来自AgentOS生成的，经过了预处理和过滤的，第一人称的感知信息，形如“我看见...我听见...我收到监护人发来的消息...”。

**近期意识流-Thinking&Act**
模型输出的内容全部由思考和工具调用组成。与思考模型的输出类似，最大的区别在于没有“回答正文”这个顶级输出类别了，取而代之的是"说话"、"发消息"等工具调用。

{Sense -> Thinking&Act}可以有多轮。

# 输出格式
Markdown，每条消息是一个顶级ATX Heading (`# `)

# 撰写线索
- 真实历史和现代人物、在真实时间点和情形下，如果AGLM是他，应该怎么想，怎么做
- 文艺作品中的人物和情景
