# DocGraph v0.1 - 简化版MVP

> **版本**：v0.1.0-alpha  
> **状态**：设计阶段  
> **目标**：快速交付可用的文档元信息聚合工具

## 版本定位

### 核心使命
提供轻量级的文档元信息管理工具，支持：
- 解析文档的YAML frontmatter
- 提取关键信息（术语、问题、依赖等）
- 自动生成汇总表格
- 实现"分散撰写，自动汇总鸟瞰"

### 设计原则
1. **技术简化**：只处理frontmatter，避免正文解析复杂性
2. **快速交付**：最小功能集，3-5天可交付
3. **用户价值优先**：先解决鸟瞰视图的核心需求
4. **渐进演进**：为后续版本预留扩展点

## 功能概述

### ✅ v0.1 核心功能
- **文件扫描**：扫描指定目录的`.md`文件
- **frontmatter解析**：仅解析YAML frontmatter（`---`之间的部分）
- **关键字段提取**：提取硬编码字段（`defines`、`issues`、`depends`等）
- **表格生成**：生成`.gen.md`后缀的汇总表格
- **基本验证**：frontmatter格式和必填字段验证

### ❌ v0.1 明确不做
- 不解析Markdown正文内容
- 不提取和分析文档链接
- 不检查双向链接
- 不处理路径规范化
- 不进行语义分析

## 技术栈

### 核心依赖
- **YAML解析**：YamlDotNet
- **文件系统**：System.IO.Abstractions
- **表格渲染**：自定义简单渲染器

### 输出约定
- **文件后缀**：`.gen.md`（明确标识机器生成）
- **文件声明**：包含"本文档由工具自动生成"提示
- **格式稳定**：相同输入产生相同输出（幂等性）

## 使用场景

### 场景1：项目文档鸟瞰
```bash
# 扫描wishes目录，生成索引
docmeta scan ./wishes --output ./wishes/index.gen.md
```

### 场景2：术语表生成
```bash
# 提取所有文档定义的术语
docmeta extract ./docs --field defines --output ./docs/glossary.gen.md
```

### 场景3：问题跟踪
```bash
# 汇总所有文档提到的问题
docmeta extract ./docs --field issues --output ./docs/issues.gen.md
```
