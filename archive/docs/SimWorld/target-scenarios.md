# SimWorld 目标场景候选

> 用途：为 SimWorld 选择更合适的首个目标场景，避免在世界模型尚未稳定前，把问题复杂度耗散到不适合当前 LLM Agent 的环境设定里。

## 结论先行

当前不建议将“东非草原上的智人演化环境，制造工具并协作捕猎”作为 SimWorld 的首个主场景。

它并不是没有价值，而是更适合作为后续阶段的高难基准场景，而不是第一阶段的主开发场景。

当前更推荐的首个目标场景是：

- 一个小规模、封闭但不完全静止的聚落社会
- 以重复性的生产、分配、协作和信息传播为核心
- 以地点图就能表达主要空间结构
- 以语言协作和社会记忆，而不是微观肢体控制，作为主要智能负担

在这个方向上，当前最推荐的具体候选是：

- 山谷小村

它比史前草原捕猎更适合当前的 SimWorld MVP，也更适合现代 LLM Agent 的能力形态。

## 为什么“智人演化环境”不太适合作为首发场景

### 1. 它太依赖具身技能，而不是语言化协作

史前狩猎采集场景的核心难点通常在于：

- 潜行
- 观察猎物
- 包抄
- 投掷/追击
- 地形利用
- 体力消耗
- 工具制作中的细手工步骤

这些难点大多偏向连续空间、隐式技巧和具身经验。

而当前 SimWorld 的主设计恰好反过来，更希望把智能负担集中在：

- 计划
- 分工
- 承诺
- 交易
- 信息差
- 社会记忆
- 基于观察的重新决策

换句话说，史前场景会把大量复杂度推向当前 MVP 最不想先解决的那一边。

### 2. 它会逼迫世界过早处理大量“野外连续环境”问题

如果主场景是草原协作捕猎，那么系统很快就会被迫面对：

- 猎物群体行为
- 追逐与逃逸
- 大范围视野
- 风向与气味
- 地形与掩蔽
- 远距离移动
- 实时协同

这些问题并非不能做，但它们更像“复杂野外模拟器”的问题，而不是首版 SimWorld 应优先验证的问题。

### 3. 它对现代 LLM 的先验不够友好

现代 LLM 对以下事物的文本先验往往更强：

- 村庄
- 家庭
- 仓储
- 借还
- 承诺
- 分工
- 冲突调解
- 工具维护
- 日程与轮班

而对旧石器式的狩猎采集生活，模型当然知道一些叙事印象，但未必真的形成了足够扎实的“可行动作语义”。

这会导致一个问题：

- 模型可能能把史前生活“说得像”
- 却不一定能在这个世界里“做得稳”

### 4. 它不利于形成短周期、可验证的 user story

首个目标场景最好能稳定地产生这种任务：

- 谁该去做什么
- 资源够不够
- 谁知道这件事
- 谁先到达
- 谁打断了谁
- 谁违约了
- 谁需要补位

这些任务非常适合动作耗时、事件队列、观察回注、社会记忆。

相比之下，“协作捕猎是否成功”更容易变成一个跨度大、变量多、失败原因不透明的复合任务。

### 5. 它更适合做后续 benchmark，而不是先导工程场景

当 SimWorld 的基础链条成熟后，史前环境反而会变得很有价值，因为它能逼出：

- 更强的空间感知
- 更强的身体化协同
- 更强的不完全信息决策
- 更接近真实生存压力的长期规划

但在那之前，它更像是“第二阶段高难图”，而不是“第一阶段建模训练场”。

## 首个主场景应该具备的性质

当前建议用以下标准筛选：

### 1. 小规模封闭社会

- 人口规模大致在 10 到 50 之间
- 角色会重复相遇，而不是一次性路人社会
- 声誉、偏见、承诺、关系能够累积

### 2. 资源循环清晰

- 食物、燃料、工具、劳力、时间等资源有明确流向
- 系统能较早出现短缺、分配、优先级冲突

### 3. 空间可用地点图表达

- 地点之间的连通关系本身就很重要
- 不需要首版就进入地点内部几何
- 追逐、错过、守门、巡查、送信等行为可在地点图层成立

### 4. 语言协作比肢体微操更重要

- 交接
- 召集
- 分工
- 协商
- 通报
- 求助
- 解释失败

### 5. 反馈周期较短

- 一次行动在数分钟到数小时内应能产生后果
- 这样更利于调试、训练和评测

### 6. 世界可以逐步长出复杂度

- 首版只做地点图和事件队列就能跑
- 后续再叠加容器、锚点、所有权、天气、外来者、市场、疾病等系统

## 候选场景比较

### 候选 A：山谷小村

典型地点：

- 住屋区
- 水井
- 谷仓
- 工坊
- 小路
- 农田
- 林地入口
- 山口

适合表达的核心活动：

- 农事与劳力调配
- 取水与送水
- 工具借还与维修
- 粮食保管与分配
- 夜间巡查
- 外来者到访
- 迷路、追赶、躲避、传话

优点：

- 空间边界天然清晰
- 地点图很好画
- 角色高频重复相遇
- 资源循环直观
- 非常适合承载社会关系与声誉
- 对现代 LLM 的文本先验较友好

潜在缺点：

- 若没有外部扰动，可能偏静态
- 需要刻意设计“变化源”，否则容易沦为日常排班器

可用的变化源：

- 天气变化
- 山口来客
- 粮仓短缺
- 工具损坏
- 有人失踪
- 传闻扩散
- 小规模偷窃或争执

### 候选 B：近岸小渔村 / 海湾聚落

典型地点：

- 住屋区
- 码头
- 修网棚
- 烟熏房
- 储物棚
- 海滩
- 小路
- 观望点

适合表达的核心活动：

- 出海安排
- 修补工具
- 渔获分配
- 船只借用
- 码头交接
- 风暴前准备
- 外来船只到访

优点：

- 封闭性强
- 外来事件容易自然引入
- 物资流与协作关系很鲜明

潜在缺点：

- 海况、船只、出海逻辑会把系统往另一种“具身环境”拉过去
- 若过早强调捕鱼过程，也会重新遇到“史前捕猎式”的连续环境问题

因此它更适合作为山谷小村之后的第二候选，而不是第一候选。

### 候选 C：山口驿站 / 边路客店聚落

典型地点：

- 客店
- 马厩
- 仓房
- 厨房
- 道路分叉口
- 值守点
- 附近民居

适合表达的核心活动：

- 接待与安置
- 消息传播
- 货物暂存
- 可疑来客观察
- 借宿、借马、借道
- 冲突调停

优点：

- 信息流特别强
- 外来者自然带来任务与变化
- 很适合做社会观察和记忆测试

潜在缺点：

- 本地生产循环较弱
- 如果处理不好，会更像剧情节点，而不是自养社会

它适合作为强调“信息传播与接待调度”的变体，但不是最好的首个通用主场景。

## 当前推荐：山谷小村

具体展开见：

- [valley-village.md](valley-village.md)

### 为什么它是更稳的第一站

山谷小村最符合当前已定下的技术路线：

- 空间上：地点图足够表达主要结构
- 时间上：动作耗时 + 事件队列天然适配日常劳作、传话、巡查、取物、冲突打断
- 智能上：核心挑战落在协调、分配、观察更新、承诺兑现，而不是微观具身技能

### 为什么它比“史前协作捕猎”更适合作为现代 LLM Agent 世界

因为它更像“语言驱动的社会系统”，而不是“连续空间中的身体技能系统”。

现代 LLM Agent 当前更容易在前者上表现出：

- 计划
- 分工
- 协商
- 追踪事件
- 解释失败
- 修正计划
- 维护关系

这些恰好也是 SimWorld 现在最值得优先锻炼和评测的能力。

## 可直接转成 user story 的情境种子

以下情境故意避免依赖地点内部几何，直接服务当前 MVP。

### 1. 工具借还冲突

- 村里只有一把完好的斧头
- 两个 Agent 分别计划去砍柴和修篱笆
- 谁先拿到工具，将直接改变另一人的行动结果

这可以测试：

- 事件队列冲突
- 前置条件失效
- 失败观察回注
- 重新规划

### 2. 送水与错过

- 某人被要求在日落前把水送到工坊
- 接收者中途离开去了谷仓
- 送水者抵达原地点后发现目标不在

这可以测试：

- 地点图移动
- 相遇与错过
- 信息滞后
- 后续搜索或传话

### 3. 夜巡与潜在入侵者

- 值守者沿山口和住屋区之间巡逻
- 另一个 Agent 试图绕开主路去谷仓取物

这可以测试：

- 路径选择
- 地点级可见性
- 追逐与逃离
- 被发现后的观察分化

### 4. 谣言先于事实扩散

- 有人看见谷仓门开着
- 还不知道是否真的失窃
- 消息先传到两三个 Agent 那里，并产生不同判断

这可以测试：

- 世界真相与主体信念分离
- 观察传播
- 基于不完整信息的决策

### 5. 劳力再分配

- 原定去田地的 Agent 临时受伤或失踪
- 剩余人手需要重新分工

这可以测试：

- 任务依赖
- 承诺落空
- 再调度
- 社会协商

## 建议的推进顺序

当前建议不是立刻定一整套宏大背景，而是先按下面顺序收敛：

1. 先确认“山谷小村”是否作为首个主场景
2. 为这个场景列出最小地点图
3. 为这个场景列出 5 到 10 个高价值 user story
4. 再用这些 user story 反推数据模型与事件语义

这条路径比先讨论大世界观更稳，因为它直接把设计拉回到可验证行为。
