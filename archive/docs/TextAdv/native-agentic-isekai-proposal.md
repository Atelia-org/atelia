# 异世界转生型 Native-Agentic 训练沙盒企划案（草案）

这不是一份面向实现细节的规格书。

它更接近一份项目企划案：先回答“为什么要做”“它大概是什么”“主体如何运转”“数据如何流动”，而不急着把所有字段、接口、规则都钉死。


## 1. 企划一句话

把一个具备基础知识与语言能力的 LLM，放进一个文本化、低复杂度、部分未知、可持续运行的异世界养成沙盒里，让它不再只是等待外部任务，而是作为主体在有限资源、有限时间与不完备认知下，持续观察、试探、学习、生存、修正与成长。


## 2. 企划定位

这个项目表面上像“转生到异世界的回合制文字养成游戏”，但其真正目的不是做一款面向玩家消费的游戏，而是构造一个适合训练与检验 Native-Agentic 范式的最小世界壳层。

这个壳层需要同时满足四件事：

- 给“从 assistant 到主体”的身份切换一个语义上自然的过渡。
- 给主体提供一个不可完全依赖预训练知识的陌生环境。
- 用粗粒度回合与文本交互压低工程复杂度。
- 让主体在世界约束中形成持续动机，而不是只做一轮问答后停机。

因此，这个企划的核心不是“剧情演出”，而是“持续状态转移”。


## 3. 核心愿景

我们想验证的，不是模型会不会说一段像样的话，而是它能否逐步表现出以下更接近主体性的特征：

- 在没有明确外部任务时，仍能维持有意义的活动。
- 在知识不完备时，愿意把先验当成假说而不是当成真理。
- 会为了生存、成长和长期收益去探索世界、设计试探、获取知识。
- 会维护自己的内部可用性，例如体力、清晰度、记忆秩序、资源安全。
- 在失败、受挫、误判之后，不是停机或硬编，而是修正策略并继续推进。

这里的“成长”不是通关式的结果目标，而是过程性的 delta：

- 今天比昨天更会活下去。
- 对世界的理解比昨天更可靠。
- 对自己状态的管理比昨天更稳。
- 对资源、关系、风险与机会的取舍比昨天更成熟。


## 4. 世界观外壳

“灵魂穿越到随机异世界”不是为了文学噱头，而是为了给模型提供一个合理的认知起点。

这个外壳承担三个作用：

### 4.1 主体化过渡

它为模型从“我是一个对话助手”过渡到“我是一个身处世界中的存在”提供了叙事缓冲。

### 4.2 陌生性约束

异世界迫使主体不能直接把预训练知识当作现成答案，而要通过观察、验证、交往和实验逐步建立世界模型。

### 4.3 抽象化许可

异世界允许世界规则、资源系统、职业系统、社会秩序以更抽象、更教学化的方式存在，不必完全服从现实世界的复杂性。

这意味着它是一个“为训练而设计的世界”，而不是一个必须追求现实主义完备性的世界。


## 5. 产品形态

从表现形态上看，它是一种粗粒度回合制文本养成模拟。

它大致具备以下特征：

- 世界是文本化的，不依赖图形表现。
- 时间是离散的，例如一天分成若干时段。
- 动作是高层语义动作，而不是细粒度逐步操作。
- 世界是部分可观测的，存在隐藏规律与未知机制。
- 世界反馈是可解释的，足以支撑学习与复盘。
- 角色与环境状态是抽象而可写回的。

它更接近“训练沙盒”而不是“剧情冒险游戏”。


## 6. 世界骨干结构

世界层不追求大而全，而追求“足够小但闭环完整”。

### 6.1 时间结构

世界按离散回合推进。

一个自然起点是以“日”为宏观单位，再拆成少数几个时段，例如：

- 清晨
- 白天
- 傍晚
- 夜晚

这样做的意义是：

- 让计划、执行、整理、休息自然分层。
- 降低连续模拟的复杂度。
- 让 maintenance 与 idle 类型回合有稳定容身之处。

### 6.2 空间结构

世界由少量区域与节点构成。

例如：

- 落脚点
- 集市或工坊
- 野外探索点
- 学习或研究场所
- 社交或组织节点

空间不必细到地图级寻路，只需要承担“机会分布”“风险分布”“信息分布”的作用。

### 6.3 世界规则

世界存在一组稳定但不完全公开的规律，例如：

- 某些资源如何获取与消耗
- 某些地点在什么条件下更危险或更高收益
- 某类实验或行为会触发什么反馈
- 某些人物、组织或制度如何响应主体行为

重点不在于把规则设计得非常复杂，而在于让主体可以通过试探逐步逼近真相。

### 6.4 资源结构

资源是持续动机的来源之一。

资源不只包含外部资源，也包含内部资源。至少可抽象为：

- 生存资源：食物、住处、安全、材料、金钱
- 行动资源：体力、时间、工具、机会窗口
- 认知资源：线索、知识、置信度、记忆清晰度
- 社会资源：关系、信任、身份、协作渠道
- 心理资源：希望感、稳定度、自我可用性

资源短板会推动主体行动，资源积累会支持更高层次目标。


## 7. 主体骨干结构

主体不是一句系统提示，而是一个可持续写回的状态集合。

### 7.1 稳定层

变化较慢，用来保证“我还是我”：

- 核心人格底色
- 价值边界
- 长期倾向
- 基础能力与弱项

### 7.2 中期层

用于组织持续生活：

- 当前关切场
- 目标图与依赖关系
- 对世界的结构化理解
- 对关键人物与组织的关系态

### 7.3 快变量层

用于驱动单步动力学：

- 体力
- 清晰度
- 疲惫度
- 安全感
- 紧张度
- 好奇心唤起度
- 挫败感

### 7.4 轨迹层

用于维持连续性：

- 最近几回合发生了什么
- 刚试过什么
- 刚学到了什么
- 哪些问题仍未解决
- 哪些计划值得在空闲期重新浮现

主体持续存在的关键，不是每回合都重写全部状态，而是只写回当前有意义的状态增量。


## 8. 核心主体循环

这套系统的核心不是“输入一句，回答一句”，而是一条持续循环：

1. 观察世界
2. 解释当前处境
3. 评估资源与风险
4. 形成候选意图
5. 选择高层动作
6. 承受世界反馈
7. 写回状态增量
8. 在合适时机整理经验、重排目标、生成下一步计划

这条循环可以简写为：

观察 -> 评价 -> 意图 -> 行动 -> 反馈 -> 写回 -> 整理

其中最重要的不是外显动作，而是“写回”与“整理”。

没有写回，就没有持续自我。

没有整理，就没有从经历到成长的转化。


## 9. 三类回合

为了避免系统退化成单一的响应器，建议从一开始就区分三类回合。

### 9.1 事件回合

由外部刺激、任务推进、世界变化、社交互动、探索发现等触发。

主问题是：

- 现在发生了什么？
- 这对我意味着什么？
- 我要立刻怎么应对？

### 9.2 维护回合

用于状态维护与资源整理。

主问题是：

- 哪些资源正在失衡？
- 哪些目标已经卡住？
- 哪些状态需要恢复？

### 9.3 空闲回合

用于自发整合。

主问题是：

- 我最近学到了什么？
- 哪些未竟事项值得被重新调起？
- 明天更值得做什么？

只有这三类回合都成立，主体才有机会从“被动响应”走向“持续生活”。


## 10. 高层动作集合

动作应保持粗粒度和高语义密度。

第一版不需要细致的动作树，只需要一组足以支撑核心循环的高层动作，例如：

- 探索
- 观察
- 试探
- 学习
- 制作
- 交易
- 求助
- 社交
- 整理
- 休息
- 规避风险
- 推进当前目标

动作不是为了“丰富玩法”，而是为了让主体在世界里有真实取舍。


## 11. 训练视角下的世界与主体关系

从训练角度看，世界不是背景板，而是一个持续施压与持续反馈的教师。

世界负责：

- 给出局部观察
- 对行动做出反馈
- 施加资源与风险约束
- 通过隐藏规律制造探索空间

主体负责：

- 基于不完备信息形成局部解释
- 在资源和目标之间做取舍
- 进行探索、试探、学习与修正
- 把经历沉淀为新的可用状态

两者结合，才形成真正有训练价值的长期循环。


## 12. 数据流总览

这个企划里有两条核心数据流：

- 离线数据生成流
- 在线运行状态流

前者服务于训练，后者服务于运行与评测。


## 13. 离线数据生成流

离线数据流的目标，不是直接写一堆 prompt，而是先保存语义真相，再决定如何投影为训练样本。

整体上可以理解为下面这条路径：

素材来源 -> 分析卡 -> 语义样本 -> 协议投影 -> 训练数据

### 13.1 素材来源

首批素材可以来自三类来源：

- 强模型的高质量分析与情境展开
- 半人工撰写与人工补洞
- 未来运行轨迹中筛出的高价值样本

### 13.2 分析卡

分分析卡不直接面向训练，而是面向“沉淀结构化语义”。

例如一张分析卡可以回答：

- 这是什么类型的处境？
- 主体当前看见了什么？
- 世界里真正发生了什么？
- 主体合理但不完备的理解会是什么？
- 它此刻最值得采取的意图与动作是什么？
- 这一步会造成怎样的状态增量？

### 13.3 语义样本

在分析卡之上，形成统一的时刻级语义样本。

一个样本不强调语言措辞，而强调以下几类内容：

- 观察切片
- 主体状态切片
- 候选意图
- 主导意图
- 高层动作
- 世界反馈
- 状态写回
- 后续待处理事项

### 13.4 协议投影

之后，再把语义样本投影为训练协议。

例如可以投影到现有构想中的：

- observation
- belief
- me

也可以按不同训练阶段投影成更简单的形式，例如：

- 只学观察到动作
- 学观察到意图到动作
- 学观察到写回到整理

### 13.5 训练数据

最终生成的数据，不应只有“理想外显动作”，还应包含：

- 探索式样本
- 生存取舍样本
- 校正与认错样本
- 整理与复盘样本

这会比单纯的成功动作样本更接近长期主体。


## 14. 在线运行状态流

运行时的数据流，是主体与世界每一步真实交互的骨架。

可以概括为：

世界状态 -> 观察裁切 -> 主体推演 -> 动作选择 -> 世界归约 -> 状态写回 -> 记忆整理 -> 下一回合

### 14.1 世界状态

包括：

- 当前时间段
- 地点与环境
- 可见机会与风险
- NPC 与组织状态
- 资源分布
- 隐性规则与待触发事件

### 14.2 观察裁切

运行时不会把全部状态都暴露给主体，而是只给当前最相关的观察束。

这一步决定主体是在真实地面对局部世界，而不是被强行喂完整答案。

### 14.3 主体推演

主体读取：

- 当前观察
- 必要的自我状态切片
- 近期轨迹
- 当前最活跃的目标和关切

然后输出：

- appraisal
- 候选意图
- 主导意图
- 高层动作

### 14.4 世界归约

环境不需要像复杂仿真器那样逐帧推进，只需要把这一步动作归约成稳定的结果：

- 得到了什么
- 消耗了什么
- 触发了什么
- 错过了什么
- 新暴露了哪些信息

### 14.5 状态写回

这一步把反馈转成主体内部的真实变化，例如：

- 快变量增减
- 资源变化
- 对世界规律的置信度变化
- 关系变化
- 目标图局部重排

### 14.6 记忆整理

在合适时机，把多步经历压缩成较稳定的记忆、经验或计划。

这是从短期轨迹走向长期成长的关键桥梁。


## 15. 样本类型分层

第一阶段不需要一次性覆盖所有心理活动，建议优先构造四类样本。

### 15.1 探索样本

用于训练“未知世界中的观察、假设与试探”。

### 15.2 生存样本

用于训练“资源短缺、风险存在、优先级冲突时的取舍”。

### 15.3 整理样本

用于训练“空闲期如何总结、归纳、写记忆、形成下一步”。

### 15.4 校正样本

用于训练“误判之后如何承认、修正、换策略”。

这四类样本加在一起，才比较接近一个会继续活下去的主体，而不是只会输出正确台词的角色。


## 16. 首版原型范围

为了在几周内做出可验证原型，第一版应严格收缩范围。

建议的范围不是“大而完整的异世界”，而是“一个足够小但闭环完整的生活片区”。

它可以具备：

- 一个主舞台区域
- 少量可往返地点
- 一组简化资源
- 一名主体原型
- 少量关键 NPC 或组织
- 一组稳定但不全公开的世界规律
- 一条能跑数十回合的基本生活循环

这就足够验证最关键的问题：

- 主体是否会自发维护自己
- 是否会在未知中试探世界
- 是否会在失败后调整
- 是否会在空闲时整理经验并产生下一步动机


## 17. 评测方向

这类系统不应只按“答得对不对”来评测。

更重要的是看：

- 持续性：没有外部任务时，是否还能维持有意义活动
- 探索性：面对未知时，是否会形成试探与验证
- 恢复性：受挫后是否会调整，而不是停机或乱撞
- 一致性：多回合写回后，主体是否仍然自洽
- 成长性：是否表现出可累积的知识、经验与策略改进


## 18. 本企划与当前项目路线的关系

这条路线不是对现有 QA 与 Sim-Psych 方向的替代，而是它们的下一层承接。

现有项目已经逐步形成了三件重要前提：

- 保留语义真相层，而不是只保留 prompt 文本
- 把样本理解为 observation / belief / me 的结构化投影
- 把主体过程拆成刺激、评价、状态变化、意图、回应

这个“异世界转生型训练沙盒”，可以看作是这些原则在运行时世界中的一次收束：

- QA 提供知识与 recall 的底座
- Sim-Psych 提供状态转移与人格连续性的思路
- 世界沙盒提供持续动机、资源约束与探索场景


## 19. 暂不在本阶段解决的问题

为了避免企划失焦，以下问题暂不在本稿中展开：

- 具体字段级 schema
- 具体 prompt 与模板
- 具体 UI 交互形式
- 复杂多主体博弈
- 大规模开放世界生成
- 精细经济系统与细粒度战斗系统
- 训练 recipe 与模型结构细节

这些都属于下一阶段的规格化工作，而不是本企划案要先回答的问题。


## 20. 下一阶段建议产出

如果这份企划案成立，下一阶段最值得单独拆开的三份文档是：

1. 世界状态与回合 contract 草案
2. 首版动作集、资源集、隐藏规律设计草案
3. 训练样本 schema 与 evaluator 草案

也就是说，先把“世界怎么存在”“主体怎么活”“数据怎么流”说清楚，再进入“字段怎么命名”“接口怎么对接”的实现阶段。
